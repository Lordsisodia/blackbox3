## Story

**As a** prospective client visiting the SISO website,
**I want** to browse SISO's portfolio by industry category,
**so that** I can quickly find relevant projects and evaluate SISO's capability in my specific industry.

---

## Acceptance Criteria

1. **AC1**: PortfolioHub page renders at `/portfolio` route with no errors
2. **AC2**: Hero section displays with gradient title ("Our Work"), subtitle, and animated waves background
3. **AC3**: Industry grid displays all 9 industries in a 3-column responsive grid (mobile: 1 col, tablet: 2 cols, desktop: 3 cols)
4. **AC4**: Each industry card shows: icon, name, project count, description, and is clickable
5. **AC5**: Clicking an industry card navigates to `/portfolio/{industry-slug}`
6. **AC6**: Featured projects section displays 3-6 recent/impressive projects using existing PortfolioCard component
7. **AC7**: Partner CTA section includes commission details (30% total) and "Become a Partner" button
8. **AC8**: Page is mobile-responsive (tested on 375px, 768px, 1920px widths)
9. **AC9**: Page load time < 2 seconds (measured with Lighthouse)
10. **AC10**: Framer Motion animations work smoothly (stagger, fade-in, scale effects)
11. **AC11**: All interactive elements have hover states (scale, border color change)
12. **AC12**: Page has SEO metadata (title, description, Open Graph tags)

### Design System Patterns to Follow

**From UX Spec (`docs/ux/portfolio-pages-ux-spec.md`):**

1. **Color Palette**:
   - Primary: `siso-orange` (#FF5722)
   - Accent: `siso-red` (#D32F2F)
   - Text: `siso-text`, `siso-text-muted`
   - Backgrounds: `siso-bg`, `siso-bg-alt`
   - Borders: `siso-border`

2. **Gradient Pattern** (for hero title):
   ```tsx
   className="bg-gradient-to-r from-siso-red to-siso-orange bg-clip-text text-transparent"
   ```

3. **Framer Motion Animation Pattern**:
   ```tsx
   // Container with stagger
   const containerVariants = {
     animate: { transition: { staggerChildren: 0.1 } }
   };

   // Item fade-in
   const itemVariants = {
     initial: { opacity: 0, y: 20 },
     animate: { opacity: 1, y: 0 }
   };

   // Card hover
   whileHover={{ scale: 1.05, transition: { duration: 0.2 } }}
   ```

4. **Responsive Grid**:
   ```tsx
   className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6"
   ```

5. **Hero Section Pattern** (from HeroSection.tsx):
   ```tsx
   <section className="min-h-[50vh] flex items-center justify-center relative overflow-hidden">
     <motion.div
       initial={{ opacity: 0, y: 20 }}
       animate={{ opacity: 1, y: 0 }}
       className="text-center space-y-6 z-10"
     >
       {/* Content */}
     </motion.div>
     <Waves {...waveProps} />
   </section>
   ```

### Featured Projects Filter Logic

```typescript
// Get featured projects (top recent/impressive)
const featuredProjects = allProjects
  .filter(p => p.featured || p.status === 'Live')
  .sort((a, b) => new Date(b.completionDate) - new Date(a.completionDate))
  .slice(0, 6);
```

### Navigation Integration

```tsx
import { useNavigate } from 'react-router-dom';

const navigate = useNavigate();

// Industry card click
onClick={() => navigate(`/portfolio/${industry.slug}`)}

// Project card click
onProjectClick={() => navigate(`/portfolio/${project.industry}/${project.slug}`)}
```

### Performance Considerations

- **Lazy Load**: Images below fold should use `loading="lazy"` attribute
- **Waves Animation**: Use existing Waves component props (already optimized)
- **Code Splitting**: Not required for this story (page is entry point)
- **Image Optimization**: Use placeholder images for now (Story 3 will add real images)

### Accessibility Requirements (WCAG 2.1 AA)

- All interactive cards have `role="button"` and `tabIndex={0}`
- Keyboard support: `onKeyDown={(e) => e.key === 'Enter' && handleClick()}`
- ARIA labels: `aria-label="View {industry.projectCount} projects in {industry.name}"`
- Focus indicators: Ensure `focus:ring-2 focus:ring-siso-orange` on all interactive elements
- Semantic HTML: Use `<nav>`, `<section>`, `<h1>`, `<h2>` appropriately
