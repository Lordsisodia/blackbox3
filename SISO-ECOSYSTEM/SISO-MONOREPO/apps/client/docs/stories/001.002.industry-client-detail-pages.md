## Story

**As a** prospective client interested in a specific industry,
**I want** to view industry-specific portfolio pages and detailed project case studies,
**so that** I can evaluate SISO's expertise in my industry and understand project scope, pricing, and deliverables.

---

## Acceptance Criteria

### Industry Landing Page (AC 1-7)

1. **AC1**: IndustryLanding page renders at `/portfolio/:industry` with industry-specific content
2. **AC2**: Breadcrumb navigation displays: "Portfolio > {Industry Name}" and is functional
3. **AC3**: Industry hero shows: industry badge, title, tagline, and stats bar (project count, avg delivery, rating)
4. **AC4**: Value proposition section displays 3 industry-specific benefits with icons
5. **AC5**: Project grid filters to show only projects from that industry
6. **AC6**: Industry CTA section encourages prospects to start (industry-tailored copy)
7. **AC7**: 404 page displays if industry slug is invalid

### Client Detail Page (AC 8-20)

8. **AC8**: ClientDetail page renders at `/portfolio/:industry/:client` with full project information
9. **AC9**: Breadcrumb navigation displays: "Portfolio > {Industry} > {Client Name}"
10. **AC10**: Project header shows: logo (if available), name, tagline, status badge, "View Live Site" button
11. **AC11**: Quick stats metadata bar displays: delivery time, investment, industry, tech stack count
12. **AC12**: Tabbed content section has tabs: Overview, Features, Screenshots, Pricing, Timeline, Testimonial (conditional)
13. **AC13**: Overview tab displays: description, tech stack grid (Frontend, Backend, Infrastructure)
14. **AC14**: Features tab shows key features, technical features, and integrations (organized lists)
15. **AC15**: Screenshots tab displays image gallery (placeholder support OK for this story)
16. **AC16**: Pricing tab shows pricing breakdown table with min/max, delivery time, currency
17. **AC17**: Timeline tab visualizes project phases with descriptions and duration
18. **AC18**: Testimonial tab displays client testimonial (only if project has one)
19. **AC19**: Related projects section shows 3 projects from same industry
20. **AC20**: Final CTA section encourages prospect to start their project

### General (AC 21-26)

21. **AC21**: Both pages are mobile-responsive (tested on 375px, 768px, 1920px)
22. **AC22**: All pages have SEO metadata (title, description, OG tags)
23. **AC23**: Page load time < 2s for both pages
24. **AC24**: Framer Motion animations work smoothly (page transitions, tab switches)
25. **AC25**: All interactive elements have hover states
26. **AC26**: Keyboard navigation and accessibility (WCAG 2.1 AA) verified

### Status Badge Color Logic

```tsx
const getStatusColor = (status: string) => {
  switch (status.toLowerCase()) {
    case 'live':
    case 'completed':
      return 'bg-green-500/20 text-green-400 border-green-500/40';
    case 'in_progress':
    case 'in development':
      return 'bg-blue-500/20 text-blue-400 border-blue-500/40';
    case 'planning':
      return 'bg-amber-500/20 text-amber-400 border-amber-500/40';
    default:
      return 'bg-gray-500/20 text-gray-400 border-gray-500/40';
  }
};
```

### Timeline Visualization Pattern

```tsx
<div className="relative">
  {/* Timeline line */}
  <div className="absolute left-4 top-0 bottom-0 w-0.5 bg-siso-border" />

  {project.timeline.phases.map((phase, index) => (
    <motion.div
      key={phase.name}
      initial={{ opacity: 0, x: -20 }}
      animate={{ opacity: 1, x: 0 }}
      transition={{ delay: index * 0.1 }}
      className="relative pl-12 pb-8"
    >
      {/* Timeline dot */}
      <div className="absolute left-2 top-1.5 w-4 h-4 rounded-full bg-siso-orange border-4 border-siso-bg" />

      {/* Phase content */}
      <div>
        <h4 className="text-lg font-semibold text-white">{phase.name}</h4>
        <p className="text-siso-text-muted text-sm">{phase.duration}</p>
        <p className="text-siso-text mt-2">{phase.description}</p>
      </div>
    </motion.div>
  ))}
</div>
```
